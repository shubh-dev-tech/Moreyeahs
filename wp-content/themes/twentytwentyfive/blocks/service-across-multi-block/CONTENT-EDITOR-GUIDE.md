# Service Across Multi Block - Content Editor Guide

## Quick Start

This block displays your multi-cloud services in a beautiful, animated layout. Perfect for showcasing cloud platforms, services, and technologies.

## Adding the Block

1. Click the **+** button in the WordPress editor
2. Search for **"Service Across Multi"**
3. Click to add the block to your page

## Filling Out the Fields

### 1. Main Heading & Description

**Main Heading** (Required)
- Enter your primary headline
- Example: "Delivering Seamless Services Across Multi-Cloud Platforms"
- Keep it concise and impactful (5-10 words)

**Main Description** (Required)
- Add supporting text that explains your services
- Example: "We enable enterprises to design, deploy, and manage secure, scalable, and high-performance solutions across leading cloud providers."
- Aim for 1-2 sentences (20-40 words)

### 2. Cloud Platforms

Click **"Add Platform"** for each cloud provider you support.

**Platform Icon** (Optional)
- Upload a logo or icon for the platform
- Recommended size: 48x48 pixels
- Format: PNG with transparent background

**Platform Name** (Required)
- Enter the full platform name
- Examples:
  - Google Cloud Platform (GCP)
  - Amazon Web Services (AWS)
  - Microsoft Azure

**Tips:**
- Add 2-4 platforms for best visual balance
- Use consistent icon sizes
- Include the most recognizable platforms first

### 3. Service Categories

Click **"Add Category"** to create service groups.

**Category Title** (Required)
- Name your service category
- Examples:
  - Core Cloud Services
  - Security & Data
  - AI, DevOps & Monitoring

**Services** (Sub-section)
- Click **"Add Service"** to list services in this category
- Enter one service per line
- Examples:
  - Software, Platform & Infrastructure Services
  - Compute Services
  - Storage Services
  - Networking Services

**Tips:**
- Create 3-4 categories for balanced layout
- Add 3-6 services per category
- Use clear, descriptive service names
- Keep service names concise (3-7 words)

### 4. Implementations & Technologies

**Implementations Section Title** (Optional)
- Default: "Implementations & Technologies"
- Customize if needed for your brand

**Technology Badges**
Click **"Add Technology"** for each tool or platform.

**Technology Name** (Required)
- Enter the technology name
- Examples: GKE, Docker, Terraform, Kubernetes

**Badge Color** (Optional)
- Choose a color for the badge background
- Default: Dark blue (#1e3a8a)
- Tip: Use brand colors for recognizable technologies

**Suggested Technologies:**
- Container Orchestration: GKE, AKS, EKS
- Infrastructure: Terraform, IaC, ARM Templates
- Monitoring: Prometheus, Grafana, ELK Stack
- DevOps: Docker, Kubernetes, Jenkins
- Networking: VPC, VPN Gateway, Cloud Load Balancer
- Storage: Cloud Storage, Big Data
- Security: IAM, Key Vault, VMSS

**Tips:**
- Add 12-20 technologies for visual impact
- Group similar technologies together
- Use recognizable abbreviations
- Vary badge colors for visual interest

### 5. Testimonial Quote

**Testimonial Quote** (Optional)
- Enter a powerful quote or statement
- Example: "The best service is delivered when customer success becomes your own."
- Keep it short and memorable (10-20 words)
- Will display with decorative quotation marks

### 6. Styling Options

**Background Color** (Optional)
- Default: Dark blue-gray (#0f172a)
- Choose a color that matches your brand
- Tip: Dark backgrounds work best for this design

**Text Color** (Optional)
- Default: White (#ffffff)
- Ensure good contrast with background
- Tip: Light text on dark background is recommended

## Best Practices

### Content Length

✅ **Do:**
- Keep headings short and punchy
- Use 1-2 sentence descriptions
- List 3-4 service categories
- Add 3-6 services per category
- Include 12-20 technology badges

❌ **Don't:**
- Write paragraph-length descriptions
- Create more than 5 service categories
- List more than 8 services per category
- Add fewer than 8 technology badges

### Visual Balance

✅ **Do:**
- Use consistent icon sizes for platforms
- Distribute services evenly across categories
- Choose complementary badge colors
- Test on mobile devices

❌ **Don't:**
- Mix different icon styles
- Create unbalanced categories (one with 2 services, another with 10)
- Use too many bright colors
- Forget to preview on mobile

### Writing Style

✅ **Do:**
- Use active voice
- Be specific and concrete
- Focus on benefits
- Use industry-standard terminology

❌ **Don't:**
- Use jargon without explanation
- Make vague claims
- Write in passive voice
- Use overly technical language

## Preview & Publish

### Before Publishing

1. **Preview the block** - Click the preview button to see how it looks
2. **Check mobile view** - Use the responsive preview options
3. **Review all text** - Check for typos and clarity
4. **Test colors** - Ensure good contrast and readability
5. **Verify links** - If you add any links in the future

### After Publishing

1. **View the live page** - Check the published version
2. **Test on mobile** - View on actual mobile devices
3. **Check animations** - Ensure smooth performance
4. **Get feedback** - Ask colleagues to review

## Troubleshooting

### Block Not Saving

- Check that all required fields are filled
- Try saving the page again
- Contact your site administrator

### Content Not Displaying

- Verify the block is not hidden
- Check that you've published (not just saved as draft)
- Clear your browser cache

### Styling Issues

- Check color contrast settings
- Verify background and text colors are set
- Try resetting to default colors

### Mobile Display Issues

- Preview in responsive mode before publishing
- Test on actual mobile devices
- Reduce content if layout looks crowded
