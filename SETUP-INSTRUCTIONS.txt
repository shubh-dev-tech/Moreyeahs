  CASE STUDY SETUP - NEXT STEPS
====================================

Your Case Study Custom Post Type has been implemented successfully!
All blocks, templates, and Next.js components are ready to use.

====================================
  QUICK START (5 MINUTES)
====================================

STEP 1: WordPress Admin Setup
------------------------------
1. Go to: http://localhost/wp-admin
2. Navigate to: Custom Fields > Tools
3. Click "Sync Available" tab
4. Sync all Case Study field groups
5. Go to: Settings > Permalinks
6. Click "Save Changes" (flushes rewrite rules)

STEP 2: Create Your First Case Study
-------------------------------------
1. Go to: Case Studies > Add New
2. Add these blocks in order:
   
   a) Case Study Header
      - Upload logo
      - Enter title (e.g., "City Dynamics")
      - Enter subtitle
      - Choose gradient colors
   
   b) Case Study Layout
      - This creates the 2-column structure
   
   c) Inside Layout - Add Left Sidebar
      - Click "Add Section"
      - Section 1: Our Client
        * Add items: Name, Role, etc.
      - Section 2: Profile
        * Add items: Location, Sector
      - Section 3: Technology
        * Add items: Tools used
   
   d) Inside Layout - Add Content Blocks
      - Meet the Client
      - The Challenges (section type: challenges)
      - The Solution (section type: solution)
      - The Outcome (section type: outcome)

3. Click "Publish"

STEP 3: Verify WordPress Frontend
----------------------------------
1. Visit: http://localhost/case-study/[your-slug]
2. Check that:
   - Header displays with gradient
   - Left sidebar shows (pink background)
   - Content renders on the right
   - Styles are applied
   - Mobile responsive works

STEP 4: Test REST API
---------------------
1. Open browser and go to:
   http://localhost/wp-json/wp/v2/case-studies
2. Should see JSON array of case studies
3. Test single:
   http://localhost/wp-json/wp/v2/case-studies/[your-slug]

STEP 5: Next.js Setup
---------------------
1. Open terminal in nextjs-wordpress folder
2. Ensure .env.local has:
   NEXT_PUBLIC_WORDPRESS_URL=http://localhost
3. Run: npm run dev
4. Visit: http://localhost:3000/case-study/[your-slug]
5. Verify all blocks render correctly

====================================
  TROUBLESHOOTING
====================================

Problem: Blocks not showing in editor
Solution: Sync ACF fields (Custom Fields > Tools > Sync)

Problem: 404 on single page
Solution: Save permalinks (Settings > Permalinks > Save)

Problem: Styles not loading
Solution: Hard refresh (Ctrl+Shift+R or Cmd+Shift+R)

Problem: Next.js can't fetch data
Solution: Check .env.local has correct WordPress URL

Problem: Sidebar not appearing
Solution: Add Left Sidebar block INSIDE the Layout block
